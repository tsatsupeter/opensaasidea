# Authentication

All API requests must include a valid API key in the `x-api-key` header.

```
x-api-key: osk_your_api_key_here
```

## Generating Keys

Create API keys from the Developer Dashboard on either site:
- [openprojectidea.com/developer-api](https://openprojectidea.com/developer-api) (Broad)
- [opensaasidea.com/developer-api](https://opensaasidea.com/developer-api) (Lite)

The same key works on both domains. Each key:

- Starts with `osk_` prefix
- Is shown **only once** at creation — store it securely
- Is hashed with SHA-256 before storage (we never store raw keys)
- Can be revoked at any time from the dashboard

<Tip>
Your first API key automatically grants **$5.00 free credit** to test the API. No credit card required.
</Tip>

## Example Request

<CodeGroup>

```bash cURL
curl -H "x-api-key: osk_abc123def456..." \
  https://opensaasidea.com/v1/api/ideas?limit=5
```

```javascript JavaScript
const res = await fetch(
  "https://opensaasidea.com/v1/api/ideas?limit=5",
  {
    headers: {
      "x-api-key": "osk_abc123def456..."
    }
  }
);
```

```python Python
import requests

res = requests.get(
    "https://opensaasidea.com/v1/api/ideas",
    headers={"x-api-key": "osk_abc123def456..."},
    params={"limit": 5}
)
```

</CodeGroup>

## Error Responses

If authentication fails, you'll receive one of these responses:

| Status | Error | Description |
|---|---|---|
| `401` | `invalid_api_key` | Missing, malformed, or unknown API key |
| `403` | `key_revoked` | The API key has been revoked |

```json 401 — Invalid API Key
{
  "error": "invalid_api_key",
  "message": "Provide a valid API key via x-api-key: osk_..."
}
```

```json 403 — Revoked Key
{
  "error": "key_revoked",
  "message": "This API key has been revoked"
}
```

<Warning>
**Never expose API keys in client-side code.** Use server-to-server requests only. If a key is compromised, revoke it immediately from the Developer Dashboard.
</Warning>
