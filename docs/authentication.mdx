All API requests must include a valid API key in the `x-api-key` header.

```bash
x-api-key: osk_your_api_key_here
```

---

## Generating Keys

<Steps>
  <Step title="Open the Developer Dashboard">
    Go to [openprojectidea.com/developer-api](https://openprojectidea.com/developer-api) and sign in.
  </Step>
  <Step title="Create a new key">
    Click **Generate API Key**. Your first key grants **$5.00 free credit** automatically.
  </Step>
  <Step title="Copy and store securely">
    The key is shown **only once**. Store it in your environment variables or a secrets manager.
  </Step>
</Steps>

<AccordionGroup>
  <Accordion title="Key properties" icon="key">
    - Starts with `osk_` prefix
    - Shown **only once** at creation - store it securely
    - Hashed with SHA-256 before storage (we never store raw keys)
    - Can be revoked at any time from the dashboard
    - Works on both `openprojectidea.com` and `opensaasidea.com` domains
  </Accordion>
</AccordionGroup>

## Example Request

<CodeGroup>

```bash cURL
curl -H "x-api-key: osk_abc123def456..." \
  https://openprojectidea.com/v1/api/ideas?limit=5
```

```javascript JavaScript
const res = await fetch(
  "https://openprojectidea.com/v1/api/ideas?limit=5",
  {
    headers: {
      "x-api-key": "osk_abc123def456..."
    }
  }
);
const { data } = await res.json();
```

```python Python
import requests

res = requests.get(
    "https://openprojectidea.com/v1/api/ideas",
    headers={"x-api-key": "osk_abc123def456..."},
    params={"limit": 5}
)
data = res.json()
```

</CodeGroup>

## Error Responses

If authentication fails, you'll receive one of these:

| Status | Error | Description |
|---|---|---|
| `401` | `invalid_api_key` | Missing, malformed, or unknown API key |
| `403` | `key_revoked` | The API key has been revoked |

<CodeGroup>

```json 401 - Invalid API Key
{
  "error": "invalid_api_key",
  "message": "Provide a valid API key via x-api-key: osk_..."
}
```

```json 403 - Revoked Key
{
  "error": "key_revoked",
  "message": "This API key has been revoked"
}
```

</CodeGroup>

<Warning>
**Never expose API keys in client-side code.** Use server-to-server requests only. If a key is compromised, revoke it immediately from the Developer Dashboard.
</Warning>
