The API uses standard HTTP status codes and returns JSON error bodies with descriptive messages.

## Error Response Format

```json
{
  "error": "error_code",
  "message": "Human-readable description of the error"
}
```

---

## Status Codes

| Code | Error | Description |
|---|---|---|
| `400` | `bad_request` | Invalid or missing query parameters |
| `401` | `invalid_api_key` | Missing, malformed, or unknown API key |
| `402` | `insufficient_credits` | Not enough credits for this request |
| `403` | `key_revoked` | The API key has been revoked |
| `404` | `not_found` | Resource not found or unknown endpoint |
| `429` | `rate_limited` | Too many requests - slow down |
| `500` | `internal_error` | Something went wrong on our end |

## Detailed Examples

<AccordionGroup>
  <Accordion title="401 - Invalid API Key" icon="shield-off">
    Returned when no API key is provided or the key is not recognized.

    ```json
    {
      "error": "invalid_api_key",
      "message": "Provide a valid API key via x-api-key: osk_..."
    }
    ```
  </Accordion>

  <Accordion title="402 - Insufficient Credits" icon="wallet">
    Returned when your credit balance is too low for the requested endpoint.

    ```json
    {
      "error": "insufficient_credits",
      "message": "Not enough API credits. Current balance: $0.03",
      "balance_cents": 3,
      "cost_cents": 2
    }
    ```
  </Accordion>

  <Accordion title="403 - Key Revoked" icon="key">
    Returned when using a key that has been revoked from the Developer Dashboard.

    ```json
    {
      "error": "key_revoked",
      "message": "This API key has been revoked"
    }
    ```
  </Accordion>

  <Accordion title="403 - Insufficient Scope" icon="lock">
    Returned when your API key lacks the required scope for the endpoint.

    ```json
    {
      "error": "insufficient_scope",
      "message": "This key lacks the 'comments:write' scope. Current scopes: [ideas:read, categories:read]"
    }
    ```
  </Accordion>

  <Accordion title="404 - Not Found" icon="search">
    Returned when requesting a resource that doesn't exist or hitting an unknown endpoint.

    ```json
    {
      "error": "not_found",
      "message": "Unknown endpoint: GET /foo",
      "docs": "https://docs.openprojectidea.com"
    }
    ```
  </Accordion>

  <Accordion title="409 - Conflict" icon="copy">
    Returned when attempting a duplicate action (e.g. bookmarking an already-bookmarked idea).

    ```json
    {
      "error": "already_bookmarked",
      "message": "Idea already bookmarked"
    }
    ```
  </Accordion>

  <Accordion title="429 - Rate Limited" icon="timer">
    Returned when you exceed the rate limit.

    ```json
    {
      "error": "rate_limited",
      "message": "Too many requests. Please wait a moment."
    }
    ```
  </Accordion>
</AccordionGroup>

<Info>
If you receive a `500` error, please retry after a few seconds. If the issue persists, contact support at **support@openprojectidea.com**.
</Info>
