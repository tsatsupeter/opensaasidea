# OpenProjectIdea API

Two APIs, one platform. Access thousands of AI-generated ideas programmatically.

- **Broad API** (`openprojectidea.com`) - Full data including revenue breakdowns, marketing strategies, SEO plans, team roles, competitors, and lead generation.
- **Lite API** (`opensaasidea.com`) - Streamlined SaaS-focused data with core fields: title, description, tech stack, pricing, and MRR estimates.

Both APIs share the same authentication, scopes, and credit system. The only difference is the depth of data returned.

## Base URLs

| Mode | Base URL | Description |
|---|---|---|
| **Broad** | `https://openprojectidea.com/v1/api` | All fields, full detail |
| **Lite** | `https://opensaasidea.com/v1/api` | SaaS-focused, streamlined |

## Quick Start

<Steps>

## Code Examples

<CodeGroup>

```bash cURL
# Broad API (full data)
curl -H "x-api-key: YOUR_API_KEY" \
  https://openprojectidea.com/v1/api/ideas

# Lite API (SaaS-focused)
curl -H "x-api-key: YOUR_API_KEY" \
  https://opensaasidea.com/v1/api/ideas
```

```javascript JavaScript
const res = await fetch(
  "https://openprojectidea.com/v1/api/ideas",
  { headers: { "x-api-key": "YOUR_API_KEY" } }
);
const { data, meta } = await res.json();
console.log(data); // array of ideas
console.log(meta); // { total, limit, offset, mode }
```

```python Python
import requests

res = requests.get(
    "https://openprojectidea.com/v1/api/ideas",
    headers={"x-api-key": "YOUR_API_KEY"},
    params={"limit": 10, "sort": "popular"}
)
data = res.json()
for idea in data["data"]:
    print(idea["title"], idea["estimated_mrr_high"])
```

</CodeGroup>
