## Two API Modes

One API key works across two domains with different data depths:

<CardGroup cols={2}>
  <Card title="Broad API" icon="layers">
    **openprojectidea.com** - Full data including revenue breakdowns, marketing strategies, SEO plans, team roles, competitors, and lead generation.
  </Card>
  <Card title="Lite API" icon="layout-list">
    **opensaasidea.com** - Streamlined SaaS-focused data with core fields: title, tech stack, pricing, and MRR estimates.
  </Card>
</CardGroup>

| Mode | Base URL |
|---|---|
| **Broad** | `https://openprojectidea.com/v1/api` |
| **Lite** | `https://opensaasidea.com/v1/api` |

<Tip>
Both modes share the same authentication, scopes, and credit system. See [API Modes](/modes) for a full field comparison.
</Tip>

## Code Examples

<CodeGroup>

```bash cURL
curl -H "x-api-key: YOUR_API_KEY" \
  "https://openprojectidea.com/v1/api/ideas?limit=10&sort=popular"
```

```javascript JavaScript
const res = await fetch(
  "https://openprojectidea.com/v1/api/ideas?limit=10&sort=popular",
  { headers: { "x-api-key": "YOUR_API_KEY" } }
);
const { data, meta } = await res.json();
console.log(`Found ${meta.total} ideas (${meta.mode} mode)`);
data.forEach(idea => console.log(idea.title, `$${idea.estimated_mrr_high}/mo`));
```

```python Python
import requests

res = requests.get(
    "https://openprojectidea.com/v1/api/ideas",
    headers={"x-api-key": "YOUR_API_KEY"},
    params={"limit": 10, "sort": "popular"}
)
result = res.json()
for idea in result["data"]:
    print(f"{idea['title']} - ${idea['estimated_mrr_high']}/mo")
```
