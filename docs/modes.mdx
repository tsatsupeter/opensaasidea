The same API key works on both domains. The difference is the **depth of data** returned in idea responses.

<CardGroup cols={2}>
  <Card title="Broad Mode" icon="layers">
    **openprojectidea.com** - Full dataset with revenue breakdowns, marketing strategies, SEO plans, team roles, and competitor analysis.
  </Card>
  <Card title="Lite Mode" icon="layout-list">
    **opensaasidea.com** - Streamlined SaaS-focused data with core fields: title, tech stack, pricing, MRR, pros and cons.
  </Card>
</CardGroup>

---

## Broad Mode

Returns the full dataset for every idea. Designed for comprehensive analysis tools and dashboards.

**Base URL:** `https://openprojectidea.com/v1/api`

<AccordionGroup>
  <Accordion title="List fields" icon="list" defaultOpen>
    `id` `title` `slug` `tagline` `category` `platform` `monetization_model` `estimated_mrr_low` `estimated_mrr_high` `vote_score` `upvotes` `downvotes` `views` `comment_count` `created_at`
  </Accordion>
  <Accordion title="Detail fields (additional)" icon="file-text">
    `description` `pricing_tiers` `estimated_daily_sales` `estimated_weekly_sales` `estimated_monthly_sales` `revenue_breakdown` `tech_stack` `team_roles` `lead_generation` `marketing_strategy` `seo_strategy` `existing_competitors` `unique_differentiators` `pros` `cons` `updated_at`
  </Accordion>
</AccordionGroup>

## Lite Mode

Returns a streamlined, SaaS-focused subset. Ideal for lightweight integrations.

**Base URL:** `https://opensaasidea.com/v1/api`

<AccordionGroup>
  <Accordion title="List fields" icon="list" defaultOpen>

  </Accordion>
  <Accordion title="Detail fields (additional)" icon="file-text">

## Field Comparison

| Field | Broad | Lite |
|---|---|---|
| Core info (title, tagline, category) | Yes | Yes |
| MRR estimates | Yes | Yes |
| Tech stack | Yes | Yes |
| Pricing tiers | Yes | Yes |
| Pros and cons | Yes | Yes |
| Revenue breakdown | Yes | - |
| Marketing strategy | Yes | - |
| SEO strategy | Yes | - |
| Team roles | Yes | - |
| Lead generation | Yes | - |
| Competitor analysis | Yes | - |
| Sales estimates (daily/weekly/monthly) | Yes | - |
| Views and comment count | Yes | Yes |
| Vote details (up/down) | Yes | Scores only |

## How Mode is Determined

The mode is automatically set based on which domain you call:

<CodeGroup>

```bash Broad Mode
curl -H "x-api-key: osk_..." \
  https://openprojectidea.com/v1/api/ideas
```

```bash Lite Mode
curl -H "x-api-key: osk_..." \
  https://opensaasidea.com/v1/api/ideas
```

</CodeGroup>

Every response includes `meta.mode` so you can verify:

```json Response
{
  "data": [...],
  "meta": {
    "total": 150,
    "limit": 20,
    "offset": 0,
    "mode": "broad"
  }
}
```

<Tip>
Non-idea endpoints (comments, votes, bookmarks, categories, stats, account) return the same data regardless of mode.
</Tip>
